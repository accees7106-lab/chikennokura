# はじめての記事

ようこそ、**わたくしのインターネット備忘録**へ。

このブログを開設してからというもの、何を最初に書くべきか悩みに悩みましたわ。技術記事にするか、日常の記録にするか、それとも自己紹介から始めるべきか……。

結局のところ、「とりあえず書き始めれば良いんですわ」という精神で、今こうしてキーボードを叩いています。

---

## わたくしについて

わたくしは、インターネットの海を漂いながら日々を生きているエンジニア（自称）ですわ。

趣味は多岐にわたりますが、主に以下のようなことを好んでいます。

- **自作キーボード** — キースイッチを変えるだけで人生が変わると信じています
- **電子工作** — ハンダゴテを握っているときが一番落ち着きますわ
- **読書** — ジャンル問わず、活字であれば何でも好きです
- **プログラミング** — 仕事でもあり趣味でもある、切っても切れない関係
- **散歩** — 気が向いたときにふらっと外へ出ます

正直なところ、どれも「本物のプロ」には程遠い、ただのインターネット好きですわ。でも、それがいいんです。

---

## なぜブログを始めたのか

正直に言うと、理由は三つあります。

### 1. 記録のため

わたくしは致命的なほどに記憶力が悪い。何かを調べて解決しても、三日後にはまた同じところで詰まっているなんてことが日常茶飯事ですわ。

「あのとき、どうやって解決したんだったかしら……」

そういう思いを何度繰り返したことか。だから、自分のための備忘録として書き残しておこうと思ったのが最初のきっかけです。

### 2. アウトプットの練習

インプットだけが増えていくと、頭の中がパンクしそうになりますわ。知識は使ってこそ定着するというのは本当で、ブログに書くことで「自分がどこまで理解できているか」が明確になります。

わかった気になっていたことが、いざ文章にしようとすると全然説明できない——そういう経験をすると、理解の甘さに気づかされますわ。

### 3. 単純に楽しそうだから

これが一番大きな理由かもしれません。

インターネット上には、素晴らしいブログがたくさんあります。毎日読んでいる技術ブログ、思わず笑ってしまう日記ブログ、写真が綺麗な散歩ブログ……。

「わたくしも、自分の城を持ちたいな」

そう思ったら、やるしかないわ、という話です。

## このサイトの技術スタック

せっかくなので、このサイト自体の構成も書き残しておきますわ。

### ホスティング

**Cloudflare Workers** を使っています。無料プランでも十分すぎるほどの機能が使えますし、何より世界中のエッジサーバーから配信されるので、どこからアクセスしても速い。

最初は GitHub Pages も検討しましたが、Cloudflare の方が設定の柔軟性が高く、速度面でも有利だったので選びました。

### 記事管理

Markdown ファイルで記事を管理しています。専用の CMS は使わず、普通のテキストエディタで `.md` ファイルを書いて、Python スクリプトでメタデータを自動生成する仕組みです。

```python
# generate_posts.py のざっくりした動き
for mdfile in posts_directory:
    metadata = parse_front_matter(mdfile)
    posts_list.append(metadata)

save_as_json(posts_list)
update_sitemap()
```

シンプルな構造ですが、これで十分です。過度なエンジニアリングは趣味ブログには要らないですわ。

### フロントエンド

- HTML / CSS / JavaScript（素のまま）
- マークダウンのレンダリングには `marked.js` を使用

フレームワークは使っていません。React や Vue のような重厚なものは必要ないですし、素の Web 技術で作るのが好きなので。

ページ遷移はすべて通常のリンクで、SPAにはしていません。シンプルが一番ですわ。

## ブログを書く上での心構え

せっかくなので、自分への戒めとして書き残しておきますわ。

### 完璧主義を捨てる

最初から完璧なブログを目指すのは諦めましたわ。最初の記事は短くていい。文章が下手でもいい。デザインが崩れていてもいい。

> 完璧を目指して一歩も動けないより、不完全でも動き続ける方がずっと良い。

これはわたくしが常日頃から自分に言い聞かせていることです。プログラムのバグ修正も同じですわ——まず動く状態にして、それから磨く。

### 継続を最優先にする

週に何本書くか、文字数はいくつ以上にする、といったルールは設けません。

書けるときに書く。書けないときは書かない。それだけです。

義務になった瞬間に楽しくなくなりますし、楽しくなければ続かない。シンプルな話ですわ。

### 読者より自分のために書く

これは少し意地悪な言い方かもしれませんが、このブログは基本的に「自分のための備忘録」です。

読んでいただける方がいれば、それは非常に嬉しいこと。でも、「読まれるために書く」ではなく、「書きたいから書く」を優先します。
