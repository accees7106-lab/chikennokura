# 自作キーボードの沼へようこそ

本日は、わたくしが愛してやまない**自作キーボード**の世界についてお話しましょう。

「キーボードなんてどれも同じでしょ？」と思っているあなた。今この瞬間がその認識を改める最後のチャンスかもしれません。自作キーボードの沼は、一度足を踏み入れたら二度と戻れないと言われています。

……もちろん、それがいいんですわ。

---

## 自作キーボードとは何か

まず基本的なところから。

市販のキーボードは完成品ですが、自作キーボードは**PCB（基板）、スイッチ、キーキャップ、ケースを自分で選んで組み立てる**ものです。

```
自作キーボードの主なパーツ
├── PCB（基板） ........... 回路が印刷された板
├── キースイッチ ......... 押したときの感触を決める部品
├── キーキャップ ......... 指で押す部分（見た目を左右する）
├── ケース ............... 基板を収めるフレーム
├── スタビライザー ....... 大きいキーがぐらつかないようにする部品
└── ファームウェア ....... キーのレイアウトを設定するソフト
```

この組み合わせが無数にあるため、**世界に一つだけの自分専用キーボード**が作れるというわけですわ。

## キースイッチの種類

自作キーボードで最も重要なパーツがキースイッチです。大きく分けると三種類あります。

### リニア軸

押し始めから底打ちまで**均一な重さ**で押せるタイプ。スコスコとした感触で、静かな打鍵音が特徴です。

代表例：Cherry MX Red、Gateron Yellow

| 特徴 | 評価 |
|-----|------|
| 打鍵音 | 静か |
| 入力のしやすさ | ◎ |
| 疲労感 | 少ない |
| タイピング精度 | △（慣れが必要） |

### タクタイル軸

キーを押す途中に**コツン**とした感触（タクタイルバンプ）がある。スイッチが反応した位置がわかりやすい。

代表例：Cherry MX Brown、Gateron Brown、Boba U4

### クリッキー軸

タクタイルバンプに加えて「カチカチ」という**クリック音**がする。打鍵感が最もわかりやすいが、うるさい。

代表例：Cherry MX Blue、Kailh Box White

> **わたくしの現在のスイッチ：Boba U4**
>
> タクタイルながら静音設計で、打鍵音がほとんど聞こえません。オフィスでも周りに迷惑をかけず、それでいて確かな打鍵感がある。今のところ最高のスイッチですわ。

## ルブ（潤滑）作業について

自作キーボード民の間で必ずと言っていいほど話題になるのが**ルブ（Lube）**です。

スイッチに潤滑剤を薄く塗ることで、摩擦を減らして滑らかな打鍵感を実現する作業のことです。

### 必要なもの

- 潤滑剤（Krytox 205g0 など）
- 小さな筆
- スイッチオープナー
- ひたすらの忍耐

### 作業の流れ

```
1. スイッチをオープナーで分解
   ↓
2. ハウジング（外側）の内壁に潤滑剤を塗る
   ↓
3. スプリングに潤滑剤を塗る（バッグルブが楽）
   ↓
4. ステムに潤滑剤を薄く塗る（タクタイル感を殺さないように）
   ↓
5. 組み立てて元に戻す
   ↓
6. 全スイッチ分（90個とか）繰り返す
```

90個のスイッチを全部ルブするのに、最初は3〜4時間かかりましたわ。でも慣れれば1時間半ほどで終わります。そして、ルブ前後の打鍵感の違いは明確で、一度やったら戻れません。

## Lily58 というキーボード

わたくしが使っている Lily58 は、**左右分割・58キー**の自作キーボードです。

### なぜ分割キーボードなのか

人間の肩幅に合わせてキーボードを配置できるため、**タイピング中の肩の開き具合が自然**になります。

普通のキーボードを使っていたころは肩こりがひどかったのですが、分割キーボードに変えてから劇的に改善されました。これだけでも自作キーボードをおすすめする価値があります。

### 58キーで足りるのか

最初は「キーが少なすぎるのでは？」と思いましたが、**レイヤー機能**で解決します。

```
レイヤー0（デフォルト）: 通常の文字入力
レイヤー1（数字・記号）: 特定キーを押しながら操作
レイヤー2（ファンクション）: F1〜F12 など
レイヤー3（マウス操作）: キーボードでマウスカーソルを動かす
```

慣れると、フルサイズキーボードよりむしろ快適になります。指の移動距離が短くなるので。

## ファームウェア：QMK について

自作キーボードのキーレイアウトは**QMK Firmware**というオープンソースのソフトウェアで設定します。

```c
// キーマップの一部（C言語で書きます）
const uint16_t PROGMEM keymaps[][MATRIX_ROWS][MATRIX_COLS] = {
  [_QWERTY] = LAYOUT(
    KC_ESC,  KC_1,  KC_2,  KC_3,  KC_4,  KC_5,
    KC_TAB,  KC_Q,  KC_W,  KC_E,  KC_R,  KC_T,
    KC_LSFT, KC_A,  KC_S,  KC_D,  KC_F,  KC_G,
    // ...
  ),
};
```

C 言語で書くので最初はとっつきにくいですが、VIA という GUI ツールを使えばコードを書かずに設定できます。
