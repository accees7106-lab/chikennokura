記事は blog/posts/ 以下の .md ファイルに書きます。
ファイル名が記事のURLになります（例: my-post.md → ?id=my-post）。


================================================================
  記事ファイルの基本構造
================================================================

ファイルの先頭に「Front Matter」と呼ばれるメタ情報を書きます。

----------------------------------------------------------------
  必須項目
----------------------------------------------------------------

---
title: 記事のタイトル
description: 一覧ページや HOME に表示される説明文（1〜2文推奨）
date: 2026-05-15
category: カテゴリ名
tags: [タグ1, タグ2, タグ3]
---

----------------------------------------------------------------
  任意項目
----------------------------------------------------------------

---
update: 2026-05-16           ← 更新日（記事ページに「更新: 」として表示）
image: /assets/img/thumb.jpg ← サムネイル画像（記事ヘッダーに表示）
---


================================================================
  基本的なMarkdown記法
================================================================

----------------------------------------------------------------
  見出し
----------------------------------------------------------------

# 見出し1（h1）← 記事本文内ではあまり使わない推奨
## 見出し2（h2）← セクション区切りに使う
### 見出し3（h3）← サブセクション
#### 見出し4（h4）← 小見出し

----------------------------------------------------------------
  文字の装飾
----------------------------------------------------------------

**太字**           → 太字になります (「」を太字にするときは半角スペースでさらに挟む)
*斜体*             → 斜体になります
~~取り消し線~~     → 取り消し線になります
`インラインコード` → 等幅フォントで赤く表示されます

----------------------------------------------------------------
  段落と改行
----------------------------------------------------------------

テキストを書くと段落になります。

空行を入れると段落が分かれます。

同じ段落内で改行したい場合は、
行末にスペースを2つ入れてから改行します。（あまり使わない）

----------------------------------------------------------------
  リスト
----------------------------------------------------------------

【箇条書き（順不同）】
- 項目1
- 項目2
  - ネストした項目（半角スペース2個でインデント）
  - ネストした項目
- 項目3

【番号付きリスト】
1. 手順1
2. 手順2
3. 手順3

  引用
----------------------------------------------------------------

> 複数行書くこともできます。
>
> 空行を入れると段落が分かれます。

----------------------------------------------------------------
  コードブロック
----------------------------------------------------------------

【言語指定なし】
```
ここにコードを書く
```

【言語指定あり（シンタックスハイライト）】
```python
def hello():
    print("Hello, World!")
```

```javascript
const greet = () => console.log('Hello!');
```

```bash
npx wrangler deploy
```

対応言語の例: python, javascript, typescript, html, css,
             bash, shell, sql, json, yaml, markdown など

================================================================
  ショートコードパーツ
================================================================

通常のMarkdown記法に加えて、独自のショートコードを使うと
リッチな表現が可能です。

----------------------------------------------------------------
  注釈ボックス（4種類）
----------------------------------------------------------------

【書き方】
::note[テキスト]
::info[テキスト]
::tip[テキスト]
::warning[テキスト]

【それぞれの用途】
::note[...]    → 青　補足情報・ノート
::info[...]    → 緑　お知らせ・仕様の説明
::tip[...]     → 黄　コツ・ヒント・豆知識
::warning[...] → 赤　注意事項・警告・エラー回避

【使用例】
::note[この記事はWindows環境を前提としています]
::info[2026年5月時点の情報です。最新情報は公式ドキュメントをご確認ください]
::tip[ショートカットキー Ctrl+S で素早く保存できます]
::warning[この操作は元に戻せません。実行前にバックアップを取ってください]

----------------------------------------------------------------
  YouTube 動画埋め込み
----------------------------------------------------------------

【書き方】
::youtube[動画ID]
::youtube[YouTubeのURL]
::youtube[youtu.beの短縮URL]

【使用例】
::youtube[dQw4w9WgXcQ]
::youtube[https://www.youtube.com/watch?v=dQw4w9WgXcQ]
::youtube[https://youtu.be/dQw4w9WgXcQ]

→ 16:9 のレスポンシブ動画プレーヤーが表示されます。
→ どの形式のURLでも自動的に動画IDを取り出します。

----------------------------------------------------------------
  リンクボタン
----------------------------------------------------------------

【書き方】
::button[ボタンに表示するテキスト | リンク先URL]

【使用例】
::button[公式ドキュメントを見る | https://developers.cloudflare.com]
::button[DLsite でダウンロード | https://www.dlsite.com]

→ 青いボタンとして表示されます。別タブで開きます。

----------------------------------------------------------------
  リンクカード
----------------------------------------------------------------

【書き方】
::card[タイトル | 説明文 | URL]

【使用例】
::card[Cloudflare Workers | サーバーレスでサイトを動かすサービス | https://workers.cloudflare.com]
::card[Supabase | オープンソースの Firebase 代替 | https://supabase.com]

→ タイトル・説明文・URLが表示される横長カードになります。
→ 外部サービスや参考ページを紹介するときに便利です。

----------------------------------------------------------------
  X（Twitter）ポストへのリンク
----------------------------------------------------------------

【書き方】
::tweet[ポストのURL]

【使用例】
::tweet[https://x.com/username/status/1234567890]

→ Xのアイコン付きリンクカードになります。

----------------------------------------------------------------
  折りたたみアコーディオン
----------------------------------------------------------------

【書き方】
:::details[クリックで展開するタイトル]
ここに詳細な内容を書きます。
Markdown記法もショートコードもそのまま使えます。
:::

【使用例】

:::details[詳しい手順を見る]
1. まず設定ファイルを開きます
2. 以下の内容を追記します

```json
{
  "key": "value"
}
```

::tip[この手順は省略することもできます]
:::

:::details[よくある質問]
**Q. エラーが出たらどうすればいいですか？**

A. まずログを確認してください。
:::

→ クリックするまで内容が折りたたまれています。
→ 長い補足情報・FAQなどを整理するのに最適です。
→ 中に他のショートコードを入れることができます（入れ子OK）。
